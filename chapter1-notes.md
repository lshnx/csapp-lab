# CS:APP 第一章笔记 — 计算机系统漫游

## 1.1 信息就是位 + 上下文

- 只由 ASCII 字符构成的文件称为 **文本文件**，所有其他文件称为 **二进制文件**
- 系统中的所有信息——磁盘文件、存储器中的程序、存储器中存放的用户数据以及网络上传送的数据——都是由一串比特（bit）表示的
- 区分不同数据对象的唯一方法是读到这些数据对象的**上下文**（context）

> 理解：同样的 0101，在不同上下文中含义完全不同。比如 4 个字节的比特串，可能是整数、浮点数、一条指令、一个地址……关键在于程序怎么解释它。

## 1.2 程序被其他程序翻译成不同的格式

hello.c 变成可执行文件的过程（四个阶段）：

| 阶段 | 工具 | 输入 | 输出 | 做了什么 |
|------|------|------|------|----------|
| 1. 预处理 | 预处理器（cpp） | hello.c | hello.i | 把 `#include` `#define` 等 # 开头的命令展开 |
| 2. 编译 | 编译器（cc1） | hello.i | hello.s | 把 C 代码翻译成汇编语言 |
| 3. 汇编 | 汇编器（as） | hello.s | hello.o | 把汇编翻译成机器指令，打包成可重定位目标文件（二进制） |
| 4. 链接 | 链接器（ld） | hello.o + 其他 .o | hello | 把多个目标文件合并成一个可执行文件 |

> 简化记忆：**预 → 编 → 汇 → 链**

## 1.3 了解编译系统如何工作是大有益处的

- **优化程序性能**：C 代码和机器指令不是一一对应的。理解编译器怎么翻译你的代码，才知道怎么写快代码
- **理解链接时出现的错误**：`undefined reference` 这类经典报错，根源在链接阶段。不懂链接原理就永远在靠猜改 bug
- **避免安全漏洞**：栈溢出攻击本质上是利用程序在内存中的表示缺陷。不懂底层就写不出安全的代码

## 1.4 处理器读并解释储存在存储器中的指令

### shell 是什么
shell 是一种命令行解释器，它输出一个提示符，等待你输入一行命令，然后执行。如果该命令行的第一个单词不是内置的 shell 命令，shell 就假设这是一个可执行文件的名字，加载并执行它。

## 1.4.1 系统的硬件组成

### 总线（Bus）
贯穿整个系统的一组电子管道，负责在各个部件之间传递信息字节。总线被设计成传送定长的字节块，也就是 **"字"（word）**。一个字中的字节数（字长）是系统的基本参数，不同系统字长不同（如 4 字节 / 8 字节）。

### I/O 设备
系统与外界的联系通道：
- 输入设备：键盘、鼠标
- 输出设备：显示器
- 存储设备：磁盘驱动器（用于长期存储数据和程序）

每个 I/O 设备通过**控制器**或**适配器**与 I/O 总线连接：
- 控制器：I/O 设备本身中或主板上的芯片组
- 适配器：插在主板插槽上的卡

> 二者功能一样：在 I/O 总线和 I/O 设备之间传递信息。区别只是封装方式不同。

### 主存（Main Memory）
处理器执行程序时用来临时存放程序和处理的数据的设备。

- 物理上：由一组 **DRAM**（动态随机存取存储器）芯片组成
- 逻辑上：一个线性的字节数组，每个字节有唯一的地址（从 0 开始），地址就是数组索引
- 组成程序的每条机器指令由不定数量的字节构成
- C 中的 `int` 类型数据在大多数机器上占 4 个字节

### 处理器（CPU）
中央处理单元，是**解释（或执行）存储在主存中指令**的引擎。

三个核心部件：
- **PC（程序计数器）**：一个字长大小的寄存器，任何时候都指向主存中的某条机器语言指令（即存放该指令的地址）
- **寄存器文件**：一个小型存储设备，由若干字长大小的寄存器组成，每个寄存器有唯一的名字
- **ALU（算术逻辑单元）**：执行算术和逻辑运算，计算新的数据和地址值

CPU 的基本工作循环（一条指令的执行过程）：
1. 从 PC 指向的内存位置读取指令
2. 解释指令中的位（判断这条指令要干什么）
3. 执行该指令指示的简单操作（加载、存储、运算、跳转等）
4. 更新 PC，指向下一条指令（下一条不一定和当前指令相邻——遇到跳转指令时 PC 会跳到别处）
