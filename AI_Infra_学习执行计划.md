## 阶段一：筑基（大一暑假 7月-8月，8 周）

### 总体目标
Linux 下能干活 · C 指针/内存不害怕 · CS:APP 到第三章 · 数据结构过一遍 · LeetCode 简单题 30-40 题

### 第 1-2 周：启动期

**只做三件事，不贪多。**

| 时间 | 任务 | 具体做什么 |
|------|------|-----------|
| 每天上午 1.5h | CS:APP 2.3-2.4 | 整数运算 + 浮点数，读一节记一节笔记 |
| 每天下午 1h | LeetCode 简单题 ×1 | 用 C 写。不挑题，按题号刷。两数之和、回文数、罗马数字这种。不会就抄答案，但要理解每一行 |
| 每天晚上 30min | Linux 环境 | 装 Ubuntu 双系统（第一天）。之后每天在终端里待一会儿：cd/ls/mkdir/touch/gcc/git |

**第 2 周结束时**：你能在终端里 `gcc hello.c && ./a.out`，知道 `segfault` 长什么样，GitHub 上有 10 天绿点。

### 第 3-4 周：C 语言强化

| 时间 | 任务 | 具体做什么 |
|------|------|-----------|
| 上午 1.5h | CS:APP 3.1-3.5 | mov/lea/算术指令/控制流。**用 `gcc -S` 看自己 C 代码的汇编**，这是你最核心的差异化能力 |
| 下午 1.5h | LeetCode + 手写 C 基础 | 数组题为主。额外：手写 `strcpy`/`strlen`/`strcmp`，不用库函数 |
| 晚上 30min | Linux | 学 gdb 基础调试。用 gdb 调试自己写的 strcpy |

**产出**：一篇"我的 C 代码对应的汇编"笔记，push 到 GitHub。

### 第 5-6 周：数据结构 + 汇编深入

| 时间 | 任务 | 具体做什么 |
|------|------|-----------|
| 上午 1.5h | CS:APP 3.6-3.10 | 过程调用/栈帧/数组/struct/缓冲区溢出 |
| 下午 1.5h | 数据结构用 C 实现 | 链表（增删改查、反转）→ 栈（数组实现+链表实现）→ 队列（循环队列） |
| 晚上 30min | 继续 Linux + gdb | 学会看 backtrace、设断点、打印变量 |

**产出**：链表/栈/队列的 C 实现，有头文件、有测试代码，push 到 GitHub。

### 第 7-8 周：数据结构继续 + Python 速通

| 时间 | 任务 | 具体做什么 |
|------|------|-----------|
| 上午 1.5h | CS:APP 第 2-3 章复习 | 做课后题，查漏补缺 |
| 下午 1.5h | LeetCode + 树 | 二叉树遍历（前中后序，递归+迭代）、二叉搜索树 |
| 晚上 30min | Python 速通 | 变量/控制流/函数/list/dict/set/文件读写。用 Python 重写你之前 C 写过的题，感受差异 |

**暑假结束时你的 GitHub 应该有的东西**：
- `csapp-notes/` — 第 1-3 章完整笔记
- `c-practice/` — 手写 C 库函数 + 数据结构实现
- `leetcode/` — 30-40 道 C 解法 + 若干 Python 解法
- 绿点覆盖整个暑假

## 阶段二：建能力（大二上 9月-1月，5 个月）

### 总体目标
数据结构完整掌握 · LeetCode 中等题能写 · 一个完整项目 · 网络/OS 基础概念

### 9月：数据结构系统化

| 任务 | 说明 |
|------|------|
| 哈希表 | 用 C 实现一个简单的 hash table（链地址法），理解 O(1) 查找 |
| 堆/优先队列 | 数组实现二叉堆，堆排序 |
| 二叉树进阶 | BST 插入删除、平衡树概念（知道 AVL/红黑树是什么就行）|
| 图的基础 | BFS/DFS、邻接表/邻接矩阵 |
| LeetCode | 继续每天一道，进入链表+树专题 |

### 10月：开始项目 + 继续刷题

**项目是你简历的核心。** 选一个，做 2-3 个月，别换。

| 项目选项 | 你学到什么 | 难度 |
|---------|-----------|------|
| **A. 用 C 写一个 HTTP 服务器** | socket、多线程、HTTP 协议解析、内存管理 | ⭐⭐⭐ |
| **B. 用 C 写一个简易数据库** | 文件 IO、B+树/哈希索引、SQL 解析 | ⭐⭐⭐ |
| **C. 简易 malloc 实现** | 内存布局、链表管理空闲块、sbrk、碎片合并 | ⭐⭐ |
| **D. 视频转码小工具（FFmpeg API）** | 音视频基础、C 工程实践、编解码概念 | ⭐⭐ |

> 我的推荐：**先 C（简易 malloc），再做 A（HTTP 服务器）**。malloc 小但硬核，HTTP 服务器完整且面试爱问。

这个月：项目立项，写设计文档，搭框架，写出第一版能跑的。

### 11月：项目打磨 + 网络基础

| 任务 | 说明 |
|------|------|
| 项目继续 | 加功能、修 bug、写测试、写 README |
| 计算机网络 | 重点：TCP/UDP 区别、三次握手四次挥手、HTTP/HTTPS、DNS。**不需要看完一整本书**，看《计算机网络自顶向下》前 3 章或 B 站图解 HTTP |
| LeetCode | 进入中等难度，回溯/贪心/DP 基础题 |

### 12月：项目收尾 + OS 基础

| 任务 | 说明 |
|------|------|
| 项目收尾 | 代码整理、README 完善（英文）、写一篇"我是怎么做的"博客 |
| 操作系统概念 | 进程 vs 线程、虚拟内存、文件系统、上下文切换。CS:APP 已经覆盖了一部分，再看《现代操作系统》前几章补充 |
| LeetCode | 保持手感，每天至少一道 |

### 1月：寒假冲刺前整理

| 任务 | 说明 |
|------|------|
| 回顾整理 | 把你暑假到现在的笔记/代码/项目理一遍 |
| 数据结构再刷一遍 | 核心 20 道题（链表反转/环形链表/有效括号/二叉树遍历/层序遍历/最大深度/路径和/在排序数组中查找/二分查找/爬楼梯/打家劫舍…） |
| CUDA 环境搭建 | 如果你有 NVIDIA 显卡，装 CUDA toolkit。没有就 Google Colab，能跑 CUDA 代码 |

## 五条原则

1. **GitHub 每天 commit。** 这可能是你简历上最有力的东西——一个大一学生，连续 365 天写代码。
2. **一次只做一件事。** C 没写熟之前不碰 Python。数据结构没看完之前不碰 CUDA。别多线程把自己搞崩。
3. **项目不在多，在完整。** 一个从头到尾写完、有 README、有测试的项目，比三个烂尾的好十倍。
4. **第一个学期不碰 CUDA。** 时候不到。地基不牢搞 GPU 编程是自虐。
5. **学校课不挂科。** 绩点对考研复试有用，重修浪费时间。
